What is GitHub?

GitHub is a free online service that allows people to use git to create code repositories, which are containers for code. A link to GitHub.


Things covered in the next section:

  • What is a branch?
  • What is a commit?
  • What is a pull request?

A branch is a section of code that has been seperated from the main branch of code, so that you can can work making changes without distrubing the main or master code. A branch should have a new and different name to the main code, every repository can one or more branches.
DO NOT MESS WITH THE MAIN BRANCH!

A commit is a save point for your code, while writing your code you can commit it to GitHub. The only helpful thing that happens once you commit multiple sections of code you make a breadcrumb trail or history of the previous code you've written, which is helpful for you and anyone else working on the project.

A pull request is a request made for the new section of code that you or someone in your group has written and would that code to be merged into the main or master code. A pull request will show up as a notification to others in the group, and then the changes proposed will be discussed, from here other people in the group can post feedback on the pull request and even add their own tweak with thier own commits.
